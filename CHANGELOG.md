# 変更履歴

このプロジェクトの主な変更は Keep a Changelog 形式で記録します。

## [Unreleased]

### Added

- 共通のページ画像候補スキャナを追加し、`manga-viewer` と `image-collector` と不要画像除外設定のプレビュー候補抽出で、lazy 属性、`srcset`、背景画像、画像リンクを共通して扱うようにしました。
- 不要画像除外設定の候補一覧にページスキャン更新ボタンを追加し、レイジーロードや仮想スクロールで後から現れる画像も登録候補へ取り込めるようにしました。
- 共通の不要画像ハッシュ除外設定を `image-collector` にも追加し、`manga-viewer` と同じ設定モーダル UI と判定処理を利用できるようにしました。
- `manga-viewer` の不要画像除外に画素ハッシュ登録を追加し、設定モーダルでプレビュー、ハッシュ詳細、登録ボタンを確認しながら除外登録できるようにしました。
- `manga-viewer` の設定モーダルに不要画像の指紋登録を追加し、登録した画像を収集対象から除外できるようにしました。
- 共通のスクリプト設定モーダルを追加し、`manga-viewer` で起動スタイルと対象サイト許可ルールを設定できるようにしました。
- `image-collector` でも共通のスクリプト設定モーダルから対象サイト許可ルールを設定できるようにしました。
- 共通 i18n 基盤 `src/shared/i18n` を追加し、ロケール検出、フォールバック、RTL 判定、欠落キー検出を利用できるようにしました。

### Changed

- `image-collector` の画像収集を共通ページ画像候補スキャナの段階的収集へ移行し、ロード済み画像と非スクロール DOM 候補を優先しつつ、候補が不足する場合だけスクロールスキャンへフォールバックするようにしました。`image-collector` を `5.6.5` に更新しました。
- 共通ページ画像候補スキャナに、ロード済み画像収集、非スクロール収集からスクロール最終フォールバックへの切り替え、候補マージ処理を追加し、`manga-viewer` の NicoManga 高速起動を共通ロジック経由にしました。`manga-viewer` を `10.13.5`、`image-collector` を `5.6.4` に更新しました。
- `manga-viewer` の NicoManga 高速起動時にロード済み画像だけを即時採用し、起動後はスクロールせず DOM 上の lazy 属性候補を追加反映するようにしました。`manga-viewer` を `10.13.4` に更新しました。
